**Implementation approach using jsPsych conditional timelines:**

jsPsych does not support inline conditional branching after a trial. Use a JavaScript variable to track attempt number and push trials onto the timeline in order, using `on_finish` to set a flag. The cleanest approach for this non-reactive context is to use jsPsych's `conditional_function` on a timeline node.

Here is the exact implementation for `buildPractice1`:

**STOP** — the above approach using `CustomEvent` and `on_load` is fragile without access to the jsPsych instance to finish the trial. Here is the correct, simple approach that DOES work:

Use jsPsych's `conditional_function` with a wrapper timeline node structure. The review button ends the trial by calling the jsPsych finishTrial — the stimulus plugin exposes this through the `jsPsych` parameter passed to `buildPractice1`.

Since `jsPsych` is already a parameter of `buildPractice1`, use it:

```typescript
export const buildPractice1 = (
  state: ExperimentState,
  updateData?: (data: DataCollection, settings: AllSettingsType) => void,
  jsPsych?: JsPsych,
): Timeline => {
  const timeline: Timeline = [];
  if (!state.isStageEnabled('practice1')) return timeline;

  let isSecondAttempt = false;
  let hadErrors = false;

  // Intro screen
  const introScreen = {
    type: htmlKeyboardResponse,
    stimulus: `
      <div class="trail-making-stage-intro">
        <p style="white-space: pre-line;">${i18n.t('TRAIL_MAKING.PRACTICE1_INTRO')}</p>
        <p class="continue-prompt">${i18n.t('TRAIL_MAKING.PRESS_TO_BEGIN')}</p>
      </div>
    `,
    choices: [' '],
  };

  // Stimulus trial factory
  const makeStimulus = (showReviewButton: boolean) => ({
    type: TrailMakingStimulusPlugin,
    stage: 'practice1',
    state,
    provide_feedback: false,
    circle_radius: state.getTrailMakingSettings().circleRadius,
    on_load: () => {
      if (!showReviewButton) return;
      // Find the jspsych display element
      const displayEl = document.getElementById('jspsych-display-element') ?? document.body;
      const reviewBtn = document.createElement('button');
      reviewBtn.id = 'practice1-review-btn';
      reviewBtn.textContent = i18n.t('TRAIL_MAKING.PRACTICE1_REVIEW_BUTTON');
      reviewBtn.style.cssText = `
        display: block;
        margin: 12px auto 0;
        padding: 10px 20px;
        font-size: 0.95em;
        cursor: pointer;
        background-color: transparent;
        color: #4a90e2;
        border: 1px solid #4a90e2;
        border-radius: 4px;
      `;
      reviewBtn.addEventListener('click', () => {
        isSecondAttempt = true;
        reviewBtn.remove();
        // Finish the current trial early so the timeline proceeds to the intro re-show + 2nd attempt
        if (jsPsych) {
          jsPsych.finishTrial({ stage: 'practice1', reviewRequested: true });
        }
      });
      displayEl.appendChild(reviewBtn);
    },
    on_finish: () => {
      if (updateData && jsPsych) {
        updateData(jsPsych.data.get(), state.getAllSettings());
      }
      // Read errors from stageResults
      const results = state.getStageResults();
      const lastResult = results[results.length - 1];
      if (lastResult) {
        hadErrors = (lastResult.errorsNonSelfCorrected + lastResult.errorsSelfCorrected) > 0;
      }
    },
  });

  // Post-practice conditional node
  const retryIntroScreen = {
    type: htmlKeyboardResponse,
    stimulus: `
      <div class="trail-making-stage-intro">
        <p style="white-space: pre-line;">${i18n.t('TRAIL_MAKING.PRACTICE1_INTRO')}</p>
        <p class="continue-prompt">${i18n.t('TRAIL_MAKING.PRESS_TO_BEGIN')}</p>
      </div>
    `,
    choices: [' '],
  };

  const retryStimulus = makeStimulus(false); // 2nd attempt: no review button

  const retryBlock = {
    timeline: [retryIntroScreen, retryStimulus],
    conditional_function: () => {
      // Show retry block if: first attempt had errors OR review was requested
      return isSecondAttempt || hadErrors;
    },
  };

  const retryMessageScreen = {
    type: htmlKeyboardResponse,
    stimulus: () => `
      <div class="trail-making-feedback">
        <p>${i18n.t('TRAIL_MAKING.PRACTICE1_RETRY_MESSAGE')}</p>
        <p class="continue-prompt">${i18n.t('TRAIL_MAKING.PRESS_TO_CONTINUE')}</p>
      </div>
    `,
    choices: [' '],
    conditional_function: () => {
      // Show "don't worry" message before the retry block
      return isSecondAttempt || hadErrors;
    },
  };

  const proceedMessageScreen = {
    type: htmlKeyboardResponse,
    stimulus: () => `
      <div class="trail-making-feedback">
        <p>${i18n.t('TRAIL_MAKING.PRACTICE1_PROCEED_MESSAGE')}</p>
        <p class="continue-prompt">${i18n.t('TRAIL_MAKING.PRESS_TO_CONTINUE')}</p>
      </div>
    `,
    choices: [' '],
  };

  timeline.push(introScreen);
  timeline.push(makeStimulus(true)); // 1st attempt: show review button
  // After 1st attempt: show retry message (conditional), then show intro+retry attempt (conditional)
  timeline.push({
    timeline: [retryMessageScreen],
    conditional_function: () => isSecondAttempt || hadErrors,
  });
  timeline.push({
    timeline: [retryIntroScreen, retryStimulus],
    conditional_function: () => isSecondAttempt || hadErrors,
  });
  // Always show proceed message at end
  timeline.push(proceedMessageScreen);

  return timeline;
};
```

**Important notes on the implementation above:**
- `conditional_function` on a timeline NODE is a jsPsych feature where the entire `timeline` array runs only if `conditional_function()` returns true. The node object must have `{ timeline: [...], conditional_function: () => boolean }`.
- The `retryMessageScreen` and `retryBlock` above use `conditional_function` as a property on a timeline object — move them into proper `{ timeline: [trial], conditional_function: ... }` wrapper objects.
- `on_finish` in the 1st stimulus trial fires AFTER the trial data is saved. At that point `state.getStageResults()` has the result for this attempt.
- When the review button is clicked, `jsPsych.finishTrial()` ends the current trial immediately. `on_finish` still fires. `isSecondAttempt` is set to `true` before `finishTrial` is called. The `hadErrors` check in `on_finish` may fire after the review click — that's fine since both `isSecondAttempt` and `hadErrors` are checked with OR.
- For `buildPractice2`: apply the exact same structure, substituting `'practice2'`, `PRACTICE2_INTRO`, `PRACTICE2_REVIEW_BUTTON`, `PRACTICE2_RETRY_MESSAGE`, `PRACTICE2_PROCEED_MESSAGE` keys and button id `'practice2-review-btn'`.

**Full timeline sequence produced:**
1. introScreen (spacebar advances)
2. makeStimulus(true) [1st attempt, review button shown]
3. `{ timeline: [retryMessageScreen], conditional_function: () => isSecondAttempt || hadErrors }`
4. `{ timeline: [retryIntroScreen, makeStimulus(false)], conditional_function: () => isSecondAttempt || hadErrors }`
5. proceedMessageScreen (always shown)

**State variable `hadErrors`:** Set inside `makeStimulus`'s `on_finish` by reading the last result from `state.getStageResults()`. Error total = `lastResult.errorsNonSelfCorrected + lastResult.errorsSelfCorrected > 0`.

**Closing note:** The old feedback screen (showing TIME_TAKEN, ERRORS_SELF_CORRECTED metrics) is REMOVED from the practice builders — replaced by the conditional retry/proceed message screens. Metrics are still recorded in data; they just don't appear on screen (per the design in D-13 and the neuropsychologist text which has no metric display).

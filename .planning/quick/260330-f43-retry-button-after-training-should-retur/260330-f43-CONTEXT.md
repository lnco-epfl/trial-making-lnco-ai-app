# Quick Task 260330-f43: retry button after training should return to beginning of instructions before retry attempt - Context

**Gathered:** 2026-03-30
**Status:** Ready for planning

<domain>
## Task Boundary

When a participant fails Practice 1 or Practice 2, the retry action should route them back to the beginning of the corresponding instruction screen before they retry the practice field.

</domain>

<decisions>
## Implementation Decisions

### Scope
- Apply to both Practice 1 and Practice 2.

### Retry Trigger
- Show retry path only when first attempt has errors.

### Retry Control
- Use an explicit retry-message button labeled "Back to instructions" instead of keyboard continue.

### the agent's Discretion
- Keep the existing retry flow structure: retry message -> instruction screen -> second practice attempt.
- Keep proceed behavior unchanged when no errors are present.

</decisions>

<specifics>
## Specific Ideas

- Keep the static practice field preview visible in the repeated instruction screen before retry.
- Add i18n key for the retry button in both EN and FR.

</specifics>

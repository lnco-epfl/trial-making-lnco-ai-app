**Changes to make:**

1. **Remove the `if (stage === 'practice1' || stage === 'practice2')` guard** — labels must appear on ALL four stages.

2. **Identify the correct first and last circles by sequence, not array index:**
   Replace:
   ```ts
   const firstCircle = layout.circles[0];
   const lastCircle = layout.circles[layout.circles.length - 1];
   ```
   With:
   ```ts
   const firstLabel = layout.sequence[0];
   const lastLabel = layout.sequence[layout.sequence.length - 1];
   const firstCircle = layout.circles.find((c) => c.label === firstLabel);
   const lastCircle = layout.circles.find((c) => c.label === lastLabel);
   ```
   Add a guard: if `!firstCircle || !lastCircle` skip label rendering.

3. **Use i18n for label text** — replace hard-coded `'START'` and `'END'` with:
   ```ts
   t('TRAIL_MAKING.START_LABEL')
   t('TRAIL_MAKING.END_LABEL')
   ```
   (These keys are added by plan 01-01. If running before 01-01, the fallback will be undefined — that is acceptable; plans should run together in wave 1 with plan 01-01 completing first or simultaneously.)

4. **Fix the label positioning** — current positions use `calc(${y}% - 50px)` above the start and `calc(${y}% + 50px)` below the end. These fixed pixel offsets break at small viewport sizes. Replace with a dynamic offset based on `circleRadius`:

   For startMarker:
   ```ts
   startMarker.style.cssText = `
     position: absolute;
     left: ${firstCircle.x}%;
     top: calc(${firstCircle.y}% + ${circleRadius + 6}px);
     transform: translateX(-50%);
     font-size: 0.75em;
     font-weight: bold;
     color: #228B22;
     z-index: 10;
     white-space: nowrap;
     pointer-events: none;
   `;
   ```

   For endMarker:
   ```ts
   endMarker.style.cssText = `
     position: absolute;
     left: ${lastCircle.x}%;
     top: calc(${lastCircle.y}% + ${circleRadius + 6}px);
     transform: translateX(-50%);
     font-size: 0.75em;
     font-weight: bold;
     color: #DC143C;
     z-index: 10;
     white-space: nowrap;
     pointer-events: none;
   `;
   ```

   Both labels appear BELOW their circle (using `circleRadius + 6` px offset), never overlapping the circle border.

5. **Move the label-appending block** to run unconditionally after all circles are added to `container` (currently at line 644 — after the `container.appendChild(circleEl)` loop). Place the label block right after the closing `});` of the circle-creation forEach, before the instruction text div is appended.

# Plan 02-01: Screen Calibration Integration

## Objective

Implement host-driven calibration by parsing `localContext.screenCalibration` and applying:

1. calibrated font size precedence on `#jspsych-display-element[data-font-size]`
2. viewport-capped `effectiveScale` to Trail Making field and circles at trial render-time
3. account-era compatibility shims (`accountId ?? memberId`) where context ownership is resolved

## Tasks

1. Add parser utility (`src/utils/screenCalibration.ts`) with strict validation:

- `fontSize`: `small|normal|large|extra-large`
- `scale`: finite number where `0.5 < scale < 3`
- returns `undefined` when no valid values

2. Read and pass calibration from `ExperimentLoader` into `run()` input.
3. Extend `run()` input type and apply calibration defaults:

- `appliedFontSize = screenCalibration?.fontSize ?? settings.generalSettings.fontSize`
- `calibrationScale = screenCalibration?.scale ?? 1`

4. Thread `calibrationScale` through `buildPractice1/2` and `buildTask1/2`.
5. Extend `TrailMakingStimulusPlugin` parameter schema with `calibration_scale` and compute:

- `progressBarHeight`
- `availableHeight`
- `maxFitScale = availableHeight / (TARGET_MAX_HEIGHT_VH/100 * window.innerHeight)`
- `effectiveScale = min(calibrationScale, maxFitScale)` (lower bounded for safety)
- apply to field dimensions and circle radius.

6. Add account-era compatibility shims in `ExperimentContext` ownership checks.
7. Update mock compatibility (`defaultMockContext.accountId`) and bump mock DB names in `main.tsx`.
8. Update Graasp dependencies to LNCO.ai forks and refresh lockfile.

## Verification

- `yarn tsc --noEmit`
- `yarn build`
- Runtime manual checks (human):
  - verify calibrated `data-font-size`
  - verify scale capping behavior on small viewport
  - verify no-regression path when calibration absent
